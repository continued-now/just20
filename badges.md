# Just20 Badges Plan

## Executive Summary

Just20 badges should feel like cute collectible stickers, not esports ranks. The goal is to make users feel proud, seen, and slightly compelled to keep collecting without making the app feel like League of Legends armor. Think soft shapes, funny names, tiny mascots, warm colors, and clear progress.

The badge system should reward four loops:

1. Streak identity: "I am becoming the kind of person who shows up."
2. Social proof: "My friends are joining because of me."
3. Performance progress: "My pushups are getting cleaner/faster/stronger."
4. Shareability: "This badge looks cute enough to post."

Badges should grant one-time XP on unlock, show progress where useful, include a Brag button for SNS sharing, and eventually show rarity like Reddit-style percentages once Just20 has enough users.

## Design Direction

Badges should look like:

- Rounded sticker badges, enamel pins, tiny patches, or soft toy medals.
- Cute mascot-led art: eggs, sparks, beans, tiny flames, floor goblins, mini trophies.
- Flat or lightly dimensional illustrations, not metallic rank plates.
- Color-coded families: streak orange, social blue, performance green, rare gold, secret charcoal.
- Friendly copy: funny, punchy, slightly teasing, never militaristic.

Avoid:

- Wings, swords, shields, skull crowns, hard rank tiers, diamond/challenger-style visuals.
- Aggressive scarcity language like "elite only" or "dominate everyone."
- Fake rarity percentages before there is enough real user data.

## Visibility Rules

Use three states:

1. Visible locked: Shows name, cute icon silhouette, requirement, XP.
2. Hidden locked: Shows "Secret Badge" with a vague hint.
3. Unlocked: Full art, unlock date, XP earned, rarity, Brag button.

## XP Economy

Badge XP should be one-time bonus XP on unlock. It should not replace workout XP.

Rules:

- Streak badge XP should be meaningful but not so large that users ignore daily XP.
- Social badge XP should be higher because referrals are growth-positive.
- Performance badge XP should reward improvement, not raw athletic ability only.
- Secret badge XP can be slightly higher because discovery feels special.
- Referral/social XP should require backend-verified users before production.

Suggested XP bands:

- Small badge: 10-60 XP
- Medium badge: 80-180 XP
- Major badge: 220-500 XP
- Legendary badge: 700-2,500 XP

Implementation detail: add XP events with source `badge_unlock` and metadata like `{ badgeId, category }`.

## Badges Screen UX

Badges should probably live inside the Streak/Profile area first, not as a bottom tab. The bottom tab bar is already full enough.

Recommended layout:

1. Header: "Badges" plus total unlocked count and total badge XP.
2. Featured next badge: the closest visible badge with progress.
3. Badge category chips: Streak, Social, Performance, Secret.
4. 3-column badge grid.
5. Badge detail modal when tapped.

Badge tile contents:

- Icon/sticker art.
- Badge name.
- XP chip.
- Locked/unlocked state.
- Tiny progress ring or progress text when applicable.
- Rarity line if available.

Badge detail modal:

- Large badge art.
- Name and description.
- Unlock requirement.
- Progress bar.
- XP reward.
- Unlock date if unlocked.
- Rarity percentage if available.
- Brag button if unlocked.
- "Next up" recommendation.

## Brag Button

Every unlocked badge should have a Brag button.

Behavior:

1. Generate a square or 9:16 share card using `react-native-view-shot`.
2. Include badge art, badge name, unlock text, Just20 brand, invite code, and optional rarity.
3. Try Instagram Stories first if available.
4. Fall back to native share sheet.

Share card variants:

- Square feed card: badge centered, clean background, invite code.
- Story card: larger badge, "I unlocked X", streak context, `#just20`.
- Social badge card: "3 friends joined because of me" with cute group art.

Do not allow bragging locked badges, except maybe "chasing this badge" later as a separate feature.

## Rarity Percentages

Rarity should be real, not simulated.

Future display examples:

- "Unlocked by 42% of users"
- "Unlocked by 8.4% of users"
- "Rare: top 3%"
- "New badge: rarity coming soon"

Backend requirement:

- `badge_unlocks` table with `badge_id`, `user_id`, `unlocked_at`.
- Aggregate `badge_unlock_count`.
- Aggregate eligible active users.
- Calculate `badge_unlock_count / eligible_user_count`.

Product rules:

- Do not show rarity until at least 500 eligible users exist.
- For very new badges, show "Rarity coming soon."
- For hidden badges, reveal rarity only after unlock.
- Avoid fake percentile copy like "Top 1%" until data is trustworthy.

## Data Model Proposal

Static badge definitions can live in code first:

```ts
type BadgeDefinition = {
  id: string;
  category: 'streak' | 'social' | 'performance' | 'consistency' | 'secret';
  name: string;
  description: string;
  requirement: string;
  xp: number;
  icon: string;
  color: string;
  visibility: 'visible' | 'hidden';
  rarityKey: string;
};
```

Local unlock table:

```sql
CREATE TABLE badge_unlocks (
  badge_id TEXT PRIMARY KEY,
  unlocked_at TEXT NOT NULL,
  xp_awarded INTEGER NOT NULL DEFAULT 0,
  share_count INTEGER NOT NULL DEFAULT 0
);
```

Backend unlock table later:

```sql
CREATE TABLE badge_unlocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id),
  badge_id TEXT NOT NULL,
  unlocked_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, badge_id)
);
```

## Unlock Engine

Run badge checks after these events:

- Workout completion.
- Streak update.
- Monthly test completion.
- Buddy linked.
- Referral accepted.
- Team room join.
- Friend completes after a nudge.

Engine should:

1. Load badge definitions.
2. Load local user stats.
3. Determine newly unlocked badges.
4. Insert into `badge_unlocks`.
5. Award XP once.
6. Queue celebration toast/modal.

## Celebration UX

Unlocks should feel delightful but not disruptive.

Recommended:

- Small bottom toast for minor badges.
- Full modal for major badges like 7, 30, 90, Popular, Evangelist.
- Haptic feedback.
- Confetti dots, not giant explosions every time.
- One clear CTA: Brag.
- Secondary CTA: View all badges.

## Implementation Phases

### Phase 1: Local Badge MVP

- Add badge definitions.
- Add local `badge_unlocks` table.
- Add badge unlock engine.
- Award local XP via existing XP system.
- Add Badges section in Streak or Squad.
- Add Brag button with native share.

### Phase 2: Better Share Cards

- Use `ViewShot` for badge-specific share cards.
- Instagram Stories fallback flow.
- Add invite/referral code to badge cards.
- Add a cute badge detail modal.

### Phase 3: Backend-Verified Social Badges

- Track referral source user.
- Track accepted referral.
- Track referred user Day 1 completion.
- Track team room joins.
- Track nudge-to-completion attribution.
- Award social badges only when backend verified.

### Phase 4: Rarity And Percentiles

- Add aggregate badge stats.
- Show rarity after enough users exist.
- Add "Rare" and "Ultra Rare" visual labels.
- Keep hidden badge rarity hidden until unlock.

## Product Cautions

- Do not overpay XP for easily faked local referrals.
- Do not show fake rarity percentages.
- Do not make the badge art too intense or competitive.
- Do not bury the core workout CTA under collectibles.
- Do not make every badge secret; users need visible goals.
- Avoid "Zealot" as a default public badge name unless the brand voice becomes more chaotic.
